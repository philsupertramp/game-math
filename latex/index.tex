\hypertarget{index_autotoc_md24}{}\doxysubsection{Extensions}\label{index_autotoc_md24}
Currently there are two different implementations of matrices/vectors \hypertarget{index_autotoc_md25}{}\doxysubsubsection{Implementation 1)}\label{index_autotoc_md25}
Game dev oriented. 2, 3 and 4 dimensional vector/matrix implementations. ~\newline
 Tightly packed data to allow minimal memory usage. see {\ttfamily ./include/math/vec} and {\ttfamily ./include/math/mat} for implementation details.\hypertarget{index_autotoc_md26}{}\doxysubsubsection{Implementation 2)}\label{index_autotoc_md26}
{\ttfamily \mbox{\hyperlink{classMatrix}{Matrix}}\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}} (see {\ttfamily ./include/math/\+Matrix.h}) represents a {\ttfamily m x n}-\/\mbox{\hyperlink{classMatrix}{Matrix}} of data of type {\ttfamily T}

a specific data science and numerics implementation. ~\newline
 Capable of Matrix-\/\+Vector and Matrix-\/\+Multiplication, it also contains data science specific algorithms and statistical methods.\hypertarget{index_autotoc_md27}{}\doxysection{Extensions}\label{index_autotoc_md27}
The library is split into multiple content seperated module. Each module is nested in the root directory \href{/include/math}{\texttt{ {\ttfamily math/}}}. ~\newline
 Each sub-\/directory contains a README file with instructions how to use the module and if it has dependencies.\hypertarget{index_autotoc_md28}{}\doxysubsubsection{Adding extensions to library}\label{index_autotoc_md28}
To activate modules use the {\ttfamily MATH\+\_\+\+EXTENSIONS} compile argument.

Example build command adding {\ttfamily numerics} module and building test suite\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{mkdir -\/p build}
\DoxyCodeLine{cmake -\/DMATH\_EXTENSIONS=numerics -\/-\/build math/build -\/-\/ -\/j 3 .}

\end{DoxyCode}
